# RFC 1870 Gap Note

`kuroshio-mta` の SMTP SIZE 実装は、受信側 SMTP セッションでの広告と `MAIL FROM` パラメータ検証を対象にしています。

## 現在カバーしている内容

- EHLO での `SIZE <max>` 広告
- `MAIL FROM` の `SIZE=` パラメータ解釈
- 不正な `SIZE` 値に対するエラー応答
- `SIZE` が上限を超えた場合の `552` 応答
- `SIZE=0` と `SIZE=max` の境界値テスト

## 実装範囲外として扱う内容

- 送信側 SMTP クライアントでの相手 `SIZE` 広告を用いた配送前最適化
- SIZE 交渉結果を配送戦略へ反映する高度な運用ロジック

## 判断メモ

README の RFC 1870 行は、受信側 SIZE 拡張の実装範囲を指します。
この範囲では主要な広告・検証・境界条件をカバーできているため、`対応済み（実装範囲内）` を維持します。
