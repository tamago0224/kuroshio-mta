# RFC 7489 Gap Note

`orinoco-mta` の DMARC 実装は、受信時評価と送信時レポート生成の基本フローを対象にしています。

## 現在カバーしている内容

- SPF / DKIM alignment 判定
- `p`, `sp`, `pct`, `aspf`, `adkim`, `fo`, `rf`, `ri`, `rua`, `ruf` の主要タグ解釈
- 組織ドメインベースのサブドメインポリシー継承
- `pct` サンプリングとポリシー適用
- 集計 / 失敗レポート生成

## 実装範囲外として扱う内容

- 外部宛先レポート配送の完全な認可検証
- XML aggregate report の受信 / 解析パイプライン
- Public Suffix List の完全実装
- すべての異常 DMARC レコードに対する厳密なエラー分類

## 判断メモ

README の RFC 7489 行は、上記の評価とレポート生成の基本範囲を指します。
RFC 7489 の運用面や周辺仕様を含む全面実装を意味するものではありません。
